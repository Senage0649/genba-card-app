# アプリ仕様書 MVP v1

## 目的
若手電気工事担当者が、現場で止まったときに**「今やること・誰に聞くか・どう伝えるか」を30秒以内**に確認できるようにする。

## 対象ユーザー
- 若手電気工事担当者
- 施工管理補助
- 現場担当者（電気工事士ではなく、施工管理・現場運営側）

## 利用シーン
- 朝礼前のスマホで「今日のシーン」を確認したい
- 現場で詰まった時に「誰に聞くか」をすぐ調べたい
- 職長に何をどう指示するか、移動中に台詞を確認したい
- 検査前に「何を準備するか」を一覧で確認したい

---

## 主要画面（5画面）

| No | 画面名 | 主な役割 |
|---:|---|---|
| 1 | ホーム画面 | 入口の集約、今日の1枚提示 |
| 2 | カード一覧画面 | 30枚を工程順で並べて見渡す |
| 3 | タグ検索画面 | タグから絞り込む |
| 4 | 困りごと別入口画面 | 7カテゴリの困りごとから逆引き |
| 5 | カード詳細画面 | 1カードの全情報を表示 |

---

## 1. ホーム画面

### 表示するもの
- **今日の1枚**：30枚から日替わりで1枚をピックアップ表示
  - 表示要素：No / title / actions（3つ）への導線
- **困りごと別入口**：7カテゴリへのボタンリスト
- **タグ検索**：人気タグを並べた入口
- **工程順で見る**：カード一覧画面への導線

### データソース
- `data/cards.json` 全件
- 「今日の1枚」は日付シードで擬似ランダムに1件選ぶ（クライアント側計算）

---

## 2. カード一覧画面

### 表示するもの（カードごと）
- **No**（id）
- **場面名**（title）
- **タグ**（tags、最大4個チップ表示）
- **一言まとめ**（summary、2行折り返し）

### 並び順
- デフォルト：**No 昇順**（工程順）
- タップでカード詳細画面に遷移

### データソース
- `data/cards.json` 全件

---

## 3. タグ検索画面

### 表示するもの
- **タグ一覧**：使用頻度順に並べる
  - 件数バッジ付き（例：`他業者調整 (16)`）
- **タグ選択時の該当カード一覧**：選んだタグを持つカードを表示
  - 表示要素：No / title / summary

### 動作
- 単一タグ選択（v1）。複数タグ AND/OR は v2 以降。

### データソース
- `data/cards.json` の `tags` フィールドを集計

---

## 4. 困りごと別入口画面

### 7カテゴリのボタン
1. **図面のどこを見ればいいかわからない**
2. **誰に聞けばいいかわからない**
3. **職長にどう指示すればいいかわからない**
4. **他業者との絡みがわからない**
5. **検査前に何を準備すればいいかわからない**
6. **怒られた理由がわからない**
7. **工事ごとの確認項目がわからない**

### 動作
- カテゴリをタップ → 該当カードリストを表示
- リスト表示要素：No / title / summary

### データソース
- カテゴリとカードのマッピング表（クライアント内に定数として保持、または `data/categories.json` を別途作成）
- 既存マッピングは `cards/index.md` の「困りごと別入口」セクションを参照

---

## 5. カード詳細画面

### 表示順（data/cards.json のフィールドを上から下に）
1. **title**（場面名・大見出し）
2. **tags**（チップ表示）
3. **actions**（今やること3つ・ナンバリング表示）
4. **materials**（まず見る資料・箇条書き）
5. **askWho**（誰に聞く？・箇条書き）
6. **askTemplate**（聞き方テンプレ・カギ括弧引用風）
7. **foremanInstruction**（職長への指示テンプレ・カギ括弧引用風）
8. **scoldedPoints**（怒られるポイント・赤系アイコン）
9. **ngActions**（NG行動・赤系アイコン）
10. **summary**（一言まとめ・最下部にカード形式で強調）

### UI 上の工夫
- **30秒読了**を意識し、`actions` と `askWho` を**ファーストビュー内**に収める
- `askTemplate` と `foremanInstruction` は**長押しでコピー可能**にする
- `scoldedPoints` / `ngActions` は赤系の警告色で目を引かせる
- スクロール末尾に**前後カードへのリンク**（< 003 ／ 005 >）

## データ仕様（data/cards.json）

各カードオブジェクトのフィールド説明：

| フィールド | 型 | 内容 | 例 |
|---|---|---|---|
| `id` | string | カード番号（3桁ゼロ埋め） | `"001"` |
| `title` | string | 場面名（# 見出し） | `"現場乗り込み前に何の資料を見ればいいかわからない"` |
| `tags` | string[] | 検索用タグ（3〜6個） | `["着工前", "図面確認", "工程確認", "他業者調整"]` |
| `actions` | string[3] | 今やること3つ（必ず3個） | `["最新版を集める", "干渉チェック", "順序を時系列で把握"]` |
| `materials` | string[] | まず見る資料 | `["施工図", "設計図", "特記仕様書"]` |
| `askWho` | string[] | 誰に聞く？ | `["現場代理人", "先輩担当者"]` |
| `askTemplate` | string[] | 聞き方テンプレ（カギ括弧付き） | `["「最新版の図面はどれですか？」"]` |
| `foremanInstruction` | string[] | 職長への指示テンプレ | `["「最新の施工図を見てください」"]` |
| `scoldedPoints` | string[] | 怒られるポイント | `["古い版次の図面で動いて手戻り"]` |
| `ngActions` | string[] | NG行動 | `["古い版次の図面で進める"]` |
| `summary` | string | 一言まとめ（1〜2文） | `"最新の図面・仕様書・工程表をデスクで揃え..."` |

### 制約
- `id` は一意。001〜030
- `actions` は厳密に3個
- 他配列は最低1個（空配列なし）
- 文字列に Markdown 記号（`#`, `##`, `-`, `**`, `` ` ``）は含まない
- 文字コード：UTF-8

## v1 でやらないこと

| 機能 | 理由 |
|---|---|
| ログイン | 検索・閲覧のみで成立する。アカウントは v2 以降 |
| ユーザー投稿 | コンテンツの品質管理コストが大きい。MVP では編集側 30 枚に絞る |
| お気に入り | ローカル保存できる v2 で実装予定 |
| 通知 | 必要な現場頻度が読めないため、利用ログを取ってから判断 |
| AIチャット | コスト・倫理レビューが MVP 範囲外 |
| 課金 | プロダクトマーケットフィット確認後に検討 |
| 画像投稿 | ストレージ・モデレーションコスト |
| 管理画面 | 30 枚は手動更新で十分。本番運用時に検討 |
| 現場別カスタマイズ | 法人契約とセットになるため後回し |
